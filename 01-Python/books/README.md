## Method per chapter

For every chapter:
1. **Predict** based on title + headings only (Law 1) — write into the chapter notebook BEFORE reading.
2. **Read actively** with notebook open. Type code by hand, don't copy-paste. Break things on purpose.
3. **Failure mode** at the end (Law 2) — describe one production scenario where this concept breaks.
4. **Summary in 5 lines** without looking back (Law 4). If you can't, you read superficially — re-read.
5. **Open questions** → move to private notes (Obsidian `pydata-book/QUESTIONS.md`).
6. **Commit + push** the notebook with a meaningful message: `"Ch XX (book): [topic] — Law 1 applied"`.

## Cadence rules

- **1 chapter every 3-5 days** depending on density. Chapters 4-10 (NumPy + pandas core) are the heaviest — expect 5-7 days each.
- **Sundays = no code.** Weekly retro instead.
- **Stuck on a concept >45 min?** Move it to QUESTIONS, finish the chapter section, return at end of week.
